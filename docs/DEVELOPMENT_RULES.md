# DEVELOPMENT_RULES.md

เอกสารนี้คือกติกาการพัฒนาเว็บแอพ **GrowDay** สำหรับใช้ควบคุมการทำงานของผู้พัฒนา, Codex, AI coding agent หรือทีมงานที่เข้ามาแก้ไขโค้ด

ไฟล์นี้ต้องใช้คู่กับเอกสารต่อไปนี้:

```txt
README.md
docs/APP_SPEC.md
docs/UI_FLOW.md
docs/DATA_SCHEMA.md
docs/API_SPEC.md
```

> เป้าหมายของไฟล์นี้คือทำให้การพัฒนาเว็บแอพ GrowDay ตรงตามขอบเขตที่ต้องการ ใช้งานได้จริง ไม่ซับซ้อนเกินไป และไม่หลุดจากแนวทาง MVP

---

## 1. หลักคิดหลักของโปรเจกต์

GrowDay เป็นเว็บแอพขนาดเล็กสำหรับกลุ่มผู้ปลูกผักไฮโดรโปนิกส์ ใช้เพื่อบันทึกการปลูก คำนวณวันเก็บเกี่ยว และบันทึกผลผลิตจริง

ระบบนี้ต้องเน้น:

1. ใช้งานง่าย
2. เปิดหน้าเว็บแล้วใช้ได้ทันที
3. รองรับมือถือเป็นหลัก
4. ไม่ต้องมีระบบ login
5. ไม่ซับซ้อนเกินจำเป็น
6. ใช้เครื่องมือฟรีหรือมีต้นทุนต่ำ
7. เหมาะกับผู้ใช้ทั่วไปที่ไม่ใช่สายเทคนิค
8. โค้ดอ่านง่าย แก้ไขต่อได้
9. ใช้ Google Sheets เป็นฐานข้อมูล
10. ใช้ Google Apps Script เป็น backend

---

## 2. ขอบเขตการพัฒนา MVP

## 8. กติกาการคำนวณวันเก็บเกี่ยว

### วันที่ปลูกหมายถึง

```txt
วันที่ย้ายต้นกล้าลงราง/ช่องปลูกไฮโดรโปนิกส์จริง
```

ไม่ใช่วันที่เพาะเมล็ด

### สูตรคำนวณ

```txt
expectedHarvestDate = plantedDate + harvestAgeDays
```

### ตัวอย่าง

```txt
plantedDate = 2026-06-05
harvestAgeDays = 25
expectedHarvestDate = 2026-06-30
```

### แหล่งอายุเก็บเกี่ยว

ต้องดึงจากชีต `Crops`

ห้าม hardcode อายุเก็บเกี่ยวไว้ใน frontend เป็นหลัก

อนุญาตให้มี fallback ใน frontend ได้เฉพาะกรณี API โหลดไม่ได้ แต่ต้องระบุชัดเจนในโค้ดว่าเป็น fallback

## 9. กติกาด้านสถานะรายการปลูก

สถานะที่ใช้ได้มีเฉพาะ:

```txt
growing
ready
harvested
cancelled
```

### ความหมาย

| Status | Label | ความหมาย |
|---|---|---|
| `growing` | กำลังปลูก | ยังไม่ถึงวันเก็บเกี่ยว |
| `ready` | พร้อมเก็บ | ถึงวันเก็บเกี่ยวแล้ว แต่ยังไม่บันทึกผลผลิต |
| `harvested` | เก็บแล้ว | บันทึกผลผลิตจริงแล้ว |
| `cancelled` | ยกเลิก | รายการถูกยกเลิก ไม่ใช้งานต่อ |

### ห้ามเพิ่ม status ใหม่โดยไม่จำเป็น

ห้ามเพิ่มสถานะเช่น:

```txt
pending
done
late
expired
active
inactive
```

ถ้าจะเพิ่ม ต้องแก้ `DATA_SCHEMA.md`, `API_SPEC.md`, `UI_FLOW.md` และโค้ดทั้งหมดที่เกี่ยวข้อง

## 10. กติกาด้าน API

ต้องยึดตาม `API_SPEC.md`

### endpoint

ใช้ Google Apps Script Web App endpoint เดียว

```txt
https://script.google.com/macros/s/{DEPLOYMENT_ID}/exec
```

### GET actions ที่ต้องรองรับ

```txt
ping
getInitialData
getCrops
getPlantings
getDashboardSummary
getReadyPlantings
```

### POST actions ที่ต้องรองรับ

```txt
createPlanting
updatePlanting
recordHarvest
cancelPlanting
seedInitialData
```

### Response format

ทุก response ต้องมี:

```json
{
  "success": true,
  "action": "actionName",
  "message": "OK",
  "data": {}
}
```

หรือกรณี error:

```json
{
  "success": false,
  "action": "actionName",
  "message": "ข้อความแจ้งผู้ใช้",
  "error": {
    "code": "ERROR_CODE",
    "details": "รายละเอียดเชิงเทคนิค"
  }
}
```

### ห้ามส่ง response แบบไม่เป็นมาตรฐาน

เช่น:

```json
["item1", "item2"]
```

หรือ

```json
{
  "ok": true
}
```

ต้องใช้รูปแบบมาตรฐานจาก `API_SPEC.md` เท่านั้น
